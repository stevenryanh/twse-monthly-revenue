# 002. 寫入用 upsert 預存程序,並全程參數化

2026-06,適用:db / api

## 背景
題目要 insert / select 兩支預存程序,且要「避免 SQL Injection」。月營收資料會重複匯入或事後補正。

## 決定
- 寫入做成 **upsert**(`MERGE` + `HOLDLOCK`),依複合主鍵:存在則更新、否則新增。
- 兩支程序**全程參數化**;API 以 `SqlParameter` 呼叫,`CommandText` 僅為程序名稱,絕無字串拼接。

## 排除了什麼
- 純 `INSERT`:重複匯入會撞主鍵而中斷,匯入不具冪等性。
- 在 API 端用字串組 SQL:直接暴露注入面,也把資料邏輯漏到應用層。

## 改動前必須想清楚的
- 若改回純 `INSERT`:重複匯入 / 補正的情境怎麼處理?冪等性由誰保證?
- 若繞過預存程序直接下 SQL:注入防線還在嗎?(`Directory.Build.props` 已把 `CA2100` 升為編譯錯誤,作為第二道防線)

## 教訓
smoke test 已驗證:同主鍵連寫兩次後仍為 1 筆、且值被更新(冪等成立)。
