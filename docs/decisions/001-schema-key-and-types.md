# 001. 資料表主鍵與欄位型別

2026-06,適用:db / api

## 背景
資料來源是證交所每月營收彙總表(t187ap05_L),每間公司每月一筆。需決定主鍵與各欄位型別,並滿足加分項「資料與原始相同」。

## 決定
- **主鍵**:複合 `(CompanyCode, DataYearMonth)` 叢集索引 — 天然唯一鍵,且讓題目要的「以公司代號查詢」直接走主鍵、最快。
- `CompanyCode` 用 `NVARCHAR(10)` — 代號可能含英文,且字串能保留前導零。
- `出表日期 / 資料年月` 用 `INT` 保留民國 YYYMMDD / YYYMM 原值,不在入庫時轉西元而失真。
- 增減 % 用 `DECIMAL(18,6)`;顯示層再四捨五入 2 位(對標題目範例)。
- 次要非叢集覆蓋索引 `IX_MonthlyRevenue_YearMonth (DataYearMonth) INCLUDE(...)` 服務「查某月全部公司」。

## 排除了什麼
- 自增 id 當主鍵 + 唯一索引:多一個無意義代理鍵,複合自然鍵已足夠。
- 公司代號用 `INT`:會吃掉前導零、無法容納含英文的代號。
- 日期存西元 `DATE`:雖規整,但與來源不同,且月報無「日」語意。

## 改動前必須想清楚的
若改主鍵或拿掉複合鍵:「同公司同月只能一筆」這個不變量由誰保證?重複匯入會不會變兩筆?
